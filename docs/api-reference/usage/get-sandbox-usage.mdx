---
title: 'Sandbox Usage'
api: 'GET /api/sandboxes/{id}/usage'
---

Per-sandbox memory usage over a window, as 1-minute points plus
envelope totals. Allocation is integrated from scale events (same
math the billing pipeline uses); used memory is sampled every 60s
from the qemu process's cgroup `memory.current`.

<Note>
v1 is memory only. CPU mirrors this shape (`cpuUsedCoreSeconds`,
`usedCpuPctAvg`, etc.) once the worker collector starts populating
cgroup `cpu.stat` — additive change, no shape break.
</Note>

<ParamField path="id" type="string" required>
  Sandbox ID.
</ParamField>

<ParamField query="from" type="string">
  RFC3339 lower bound. Default: now minus 1 hour.
</ParamField>

<ParamField query="to" type="string">
  RFC3339 upper bound. Default: now. Window must be ≤ 30 days.
</ParamField>

### Units

All `*GbSeconds` fields are **GiB-seconds** (binary, 2<sup>30</sup>
bytes/GiB). For a sandbox provisioned at 1024 MiB running for one
minute, `memoryAllocatedGbSeconds` is `1024/1024 × 60 = 60`. For an
RSS sample of 612 MiB, `memoryUsedGbSeconds` is `612/1024 × 60 ≈
35.86`. The wire name uses `Gb` for backwards compatibility with the
billing aggregator; do not interpret it as decimal GB.

### Reading the response

`totals.memoryAllocatedGbSeconds` is what the bill is computed from —
the same physical quantity the Stripe pipeline reports. `totals.memoryUsedGbSeconds`
is the integrated actual RSS; the ratio between the two tells you
whether the sandbox is over-provisioned.

Each point's `memoryAllocatedGbSeconds` and `memoryUsedGbSeconds`
integrate over that 1-minute bucket — summing them across any
subrange reconstructs the integral for that subrange. `totals.*` is
the server-side sum across all returned points. Snapshot scalars
(`allocatedMemoryMb`, `usedMemoryMb{Avg,Peak}`) are time-weighted
averages over the bucket and are intended for chart rendering.

### Bucket semantics

- Buckets are minute-aligned in UTC, not aligned to `from`. The first
  and last buckets may be partial if `from` / `to` are mid-minute.
- Minutes where the sandbox is not running emit zero-points
  (`uptimeSeconds: 0`, all integrals `0`) rather than nulls — gaps
  read as continuous flat zero on a chart.
- When a sandbox is resized mid-bucket, `allocatedMemoryMb` shows the
  time-weighted average across the bucket; subsequent buckets reflect
  the new tier.

### Use the aggregator for cross-sandbox questions

To rank sandboxes by spend (or, soon, by actual usage), use
[`GET /api/usage`](/api-reference/usage/get-usage) — that endpoint
serves cross-sandbox aggregates and supports filter/sort/cursor. The
per-sandbox endpoint here is for "look at this one sandbox in detail."
